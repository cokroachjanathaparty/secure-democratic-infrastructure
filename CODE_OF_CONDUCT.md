
# Code of Conduct

# Secure Democratic Infrastructure (SDI)

Secure Democratic Infrastructure (SDI) is an open constitutional technology initiative focused on building transparent, secure, auditable, and privacy-preserving democratic infrastructure.

We are committed to maintaining an open, respectful, collaborative, and security-conscious community.

---

# Our Commitment

We aim to create an environment that is:

- respectful,
- inclusive,
- technically rigorous,
- security-focused,
- collaborative,
- and constructive.

Contributors, maintainers, researchers, and participants are expected to work together professionally and respectfully.

---

# Expected Behavior

Participants are expected to:

- Be respectful and professional
- Engage in constructive technical discussion
- Respect differing viewpoints and expertise
- Prioritize evidence-based engineering decisions
- Accept technical review and feedback professionally
- Focus on collaboration and public benefit
- Support responsible disclosure practices
- Maintain transparency and integrity

---

# Security-First Responsibility

Because SDI involves security-critical infrastructure research, contributors are expected to:

- Avoid reckless disclosure of vulnerabilities
- Follow responsible disclosure practices
- Respect security review processes
- Avoid intentionally harmful contributions
- Prioritize public safety and infrastructure integrity

---

# Unacceptable Behavior

The following behaviors are not tolerated:

- Harassment or intimidation
- Personal attacks
- Hate speech or discrimination
- Malicious code contributions
- Intentional security compromise attempts
- Public disclosure of unpatched vulnerabilities
- Spam or disruptive behavior
- Fraudulent or deceptive activity
- Doxxing or privacy violations

---

# Technical Discussion Standards

Technical disagreements are expected in research-driven projects.

Discussions should remain:

- respectful,
- evidence-based,
- technically focused,
- and professionally conducted.

---

# Contribution Integrity

Contributors should:

- Submit original work or properly attributed work
- Avoid plagiarism
- Avoid hidden functionality
- Avoid malicious dependencies
- Maintain transparent engineering practices

---

# Responsible Disclosure

Security vulnerabilities should be reported privately following the procedures described in:

`SECURITY.md`

Do not publicly disclose vulnerabilities before coordinated review.

---

# Enforcement Responsibilities

Project maintainers may:

- Review reports of misconduct
- Remove inappropriate content
- Restrict repository access
- Reject harmful contributions
- Temporarily or permanently ban participants violating this policy

---

# Scope

This Code of Conduct applies to:

- GitHub discussions
- Pull requests
- Issues
- RFC discussions
- Documentation collaboration
- Community communication channels
- Security coordination discussions

---

# Community Values

SDI values:

- Open collaboration
- Constitutional transparency
- Public accountability
- Security-first engineering
- Privacy preservation
- Research integrity
- Long-term public trust

---

# Reporting Issues

If you experience or witness unacceptable behavior, report it to the maintainers privately through appropriate repository communication channels.

Reports should include:

- Description of the incident
- Relevant evidence if available
- Context of the interaction
- Impact assessment if relevant

---

# Good Faith Participation

We encourage:

- constructive criticism,
- responsible debate,
- peer review,
- academic collaboration,
- and security research conducted in good faith.

---

# Strategic Goal

The SDI community exists to support the development of transparent, auditable, secure, privacy-preserving democratic infrastructure through open collaboration, responsible engineering, and distributed constitutional trust.
